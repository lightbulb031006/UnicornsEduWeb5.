# Issue Tracker: Local Markdown

Issues for this repo live as markdown files under `.scratch/` by default.

Use GitHub Issues only when the user explicitly asks to publish externally.

## Conventions

- Create issues as `.md` files under `.scratch/<feature-or-topic>/`.
- Use numbered filenames in dependency order, for example `001-fix-range-and-note-clearing.md`.
- Put the triage label near the top of each file, for example `Labels: needs-triage`.
- Keep issue bodies in the standard issue template used by the relevant skill.
- Do not upload local issue contents to an external tracker unless explicitly requested.

## When A Skill Says "Publish To The Issue Tracker"

Write a local markdown issue under `.scratch/` and include the requested triage label in the file.

## When A Skill Says "Fetch The Relevant Ticket"

Read the relevant markdown file from `.scratch/` unless the user provides a different tracker link.
